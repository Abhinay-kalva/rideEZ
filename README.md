# RideEZ - Bike Pooling & Ride Sharing Platform

A full-stack bike pooling web application similar to BlaBlaCar, focused on two-wheelers. The platform connects bike owners offering rides with people looking for a ride on the same route.

## ğŸš€ Features

- **User Authentication**: Sign up, login with phone/email, JWT-based authentication
- **Role-based Access**: Rider (Driver) and Passenger roles
- **Ride Management**: Create, search, and manage rides
- **Booking System**: Book available rides, prevent double booking
- **Real-time Tracking**: Live GPS location updates using Firebase Realtime Database
- **Google Maps Integration**: Route planning, navigation, and location visualization
- **Ratings & Reviews**: Rate riders after ride completion
- **Safety Features**: SOS button, live ride sharing

## ğŸ—„ Database Setup

1. **Create MySQL Database**:
   ```sql
   CREATE DATABASE rideez_db;
   ```

2. **Update Database Credentials**:
   Edit `backend/src/main/resources/application.properties`:
   ```properties
   spring.datasource.username=your_username
   spring.datasource.password=your_password
   ```

   The schema will be auto-generated by Hibernate on first run.

   Alternatively, you can run the SQL script manually:
   ```bash
   mysql -u your_username -p rideez_db < backend/src/main/resources/schema.sql
   ```

## ğŸ”¥ Firebase Setup

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Create a new project or select an existing one
3. Enable **Realtime Database**
4. Copy your Firebase configuration
5. Update `frontend/js/config.js` with your Firebase config:
   ```javascript
   FIREBASE_CONFIG: {
       apiKey: "YOUR_API_KEY",
       authDomain: "YOUR_AUTH_DOMAIN",
       databaseURL: "YOUR_DATABASE_URL",
       projectId: "YOUR_PROJECT_ID",
       storageBucket: "YOUR_STORAGE_BUCKET",
       messagingSenderId: "YOUR_MESSAGING_SENDER_ID",
       appId: "YOUR_APP_ID"
   }
   ```

6. Set Realtime Database rules (for development):
   ```json
   {
     "rules": {
       "rides": {
         ".read": true,
         ".write": true
       }
     }
   }
   ```

## ğŸ—º Google Maps Setup

1. Get your API key from [Google Cloud Console](https://console.cloud.google.com/)
2. Enable the following APIs:
   - Maps JavaScript API
   - Places API
   - Directions API
   - Geocoding API
3. Update `frontend/index.html`:
   Replace `YOUR_GOOGLE_MAPS_API_KEY` with your actual API key in the script tag.
4. Update `frontend/js/config.js`:
   ```javascript
   GOOGLE_MAPS_API_KEY: "YOUR_GOOGLE_MAPS_API_KEY"
   ```

## ğŸš€ Running the Application

### Backend (Spring Boot)

1. **Navigate to backend directory**:
   ```bash
   cd backend
   ```

2. **Build the project**:
   ```bash
   mvn clean install
   ```

3. **Run the application**:
   ```bash
   mvn spring-boot:run
   ```

   Or run the main class:
   ```bash
   java -jar target/rideez-backend-1.0.0.jar
   ```

   The backend will start on `http://localhost:8080`

### Frontend

You have several options to serve the frontend:

#### Option 1: Using Python HTTP Server
```bash
cd frontend
python -m http.server 8000
```

#### Option 2: Using Node.js HTTP Server
```bash
cd frontend
npx http-server -p 8000
```

#### Option 3: Using VS Code Live Server
- Install "Live Server" extension
- Right-click on `index.html` and select "Open with Live Server"

#### Option 4: Using any Web Server
- Place the `frontend` folder in your web server's document root
- Access via browser

The frontend will be available at `http://localhost:8000` (or your configured port)

## ğŸ¯ API Endpoints

### Authentication
- `POST /auth/register` - Register new user
- `POST /auth/login` - Login user

### Rides
- `POST /rides/create` - Create a new ride (Rider only)
- `GET /rides/search` - Search available rides
- `GET /rides/my-rides` - Get rider's rides
- `GET /rides/{id}` - Get ride by ID
- `POST /rides/{id}/start` - Start a ride (Rider only)
- `POST /rides/{id}/end` - End a ride (Rider only)

### Bookings
- `POST /bookings/book/{rideId}` - Book a ride (Passenger only)
- `GET /bookings/my-bookings` - Get passenger's bookings
- `GET /bookings/ride/{rideId}` - Get bookings for a ride

### Ratings
- `POST /ratings/add` - Add rating for a completed ride

All authenticated endpoints require `Authorization: Bearer <token>` header.

## ğŸ› Troubleshooting

### Backend won't start
- Check MySQL is running
- Verify database credentials in `application.properties`
- Check if port 8080 is available

### Maps not loading
- Verify Google Maps API key is correct
- Check browser console for errors
- Ensure required APIs are enabled in Google Cloud Console

### Firebase not working
- Verify Firebase config in `config.js`
- Check Firebase Realtime Database is enabled
- Verify database rules allow read/write

### CORS errors
- Ensure backend CORS configuration includes your frontend URL
- Check browser console for specific CORS errors
